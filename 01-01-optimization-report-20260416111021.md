## 1. 本次优化目标

基于 `01-project-review-20260416105600.md` 中的高优先级问题，本次优化聚焦以下几个可直接落地且收益最高的方向：

1. 收紧 API Key 配置，避免默认弱密钥直接进入生产环境。
2. 明确 `prod` profile 边界，关闭 H2 Console 和 Demo Seed。
3. 修复订单号按记录数生成导致的并发和准确性风险。
4. 收紧 MCP `lineItems` 参数解析，避免过宽异常捕获。
5. 补齐针对上述行为的自动化测试。

## 2. 已完成优化

### 2.1 API Key 配置与鉴权响应

- 将单值 `enterprise.security.api-key` 调整为多值 `enterprise.security.api-keys`，支持逗号分隔的密钥轮换。
- 新增 `enterprise.security.allow-default-api-key` 开关。
- 默认 profile 使用 `ENTERPRISE_API_KEYS` 环境变量；本地未设置时回落到 `dev-local-api-key`，便于开箱运行。
- `prod` profile 禁止默认本地密钥，未显式提供 `ENTERPRISE_API_KEYS` 时将直接阻止启动。
- `ApiKeyAuthenticationFilter` 改为基于 `ObjectMapper` 输出标准 JSON 错误体，避免手工 JSON 转义问题。
- API Key 比对增加规范化与常量时间比较，降低错误配置和简单侧信道风险。

### 2.2 生产 Profile 收紧

- 新增 `src/main/resources/application-prod.yml`。
- 在 `prod` profile 中关闭 `spring.h2.console.enabled`。
- 在 `prod` profile 中关闭 `enterprise.demo.seed-enabled`。
- 在 `prod` profile 中关闭 `management.info.env.enabled`，并将健康检查详情调整为 `when_authorized`。
- `SecurityConfig` 和 `ApiKeyAuthenticationFilter` 仅在 H2 Console 显式开启时才放行 `/h2-console/**`。

### 2.3 订单号生成改造

- 新增表 `order_number_sequences` 与 Flyway 迁移 `V2__order_number_sequences.sql`。
- 新增 `OrderNumberSequence` / `OrderNumberSequenceRepository`。
- `SalesOrderService.nextOrderNo()` 不再依赖 `findAll().size()`，改为数据库行级锁下分配流水号。
- 新方案可避免并发创建订单时的重复编号和因删除历史订单导致的编号回退问题。

### 2.4 MCP 参数校验收紧

- `EnterpriseMcpServer.parseLineItems()` 改为先做 token 结构校验，再做数量格式校验。
- 对 `SKU:QTY` 格式错误、空值、非数字数量、大整数溢出分别给出更具体的业务错误信息。
- 避免过去由整段 `try/catch NumberFormatException` 带来的误判和模糊报错。

### 2.5 全局异常处理补强

- `GlobalExceptionHandler` 新增 `NoResourceFoundException` 处理。
- 对于未命中的静态资源或关闭后的 `/h2-console/`，现在返回 `404` 而不是兜底 `500`。
