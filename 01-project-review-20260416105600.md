## 一、项目概述

`gi025` 是一个基于 **Spring Boot 4.0.5** 和 **Spring AI 2.0.0-M4** 搭建的企业级基础底座项目。项目定位为“可直接继续扩展的基础盘”，而非最小 demo。它整合了传统 REST API、AI Chat 能力以及最新的 MCP（Model Context Protocol）Server/Client 能力，目标客户/产品/订单/知识库四大业务领域。

## 五、主要优点

1. **技术选型前瞻**：采用 Spring Boot 4 + Spring AI 2.0.0-M4 + MCP Streamable-HTTP，处于技术前沿。
2. **README 质量高**：不仅说明了怎么跑，还解释了“为什么这样选”，并附上了所有官方文档链接。
3. **配置设计合理**：默认关闭 AI 模型和 MCP Client，确保无外部依赖时也能直接启动。
4. **统一响应与异常处理**：`ApiResponse`、`ApiErrorResponse` 配合 `GlobalExceptionHandler`，REST 接口输出规范一致。
5. **请求链路追踪**：`RequestIdFilter` 自动生成/透传 `X-Request-Id`，并写入 MDC，便于日志排查。
6. **MCP 能力完整**：同时暴露了 Tools、Resources（含模板）、Prompts 三类能力，Annotation 驱动实现简洁。
7. **Demo 数据即开即用**：`DemoDataInitializer` 在空库时自动注入客户、产品、知识库和一笔种子订单。

## 六、存在的问题与改进建议

### 6.1 安全问题（建议优先处理）

| 问题 | 风险 | 建议 |
|------|------|------|
| API Key 硬编码在 `application.yml` | 密钥泄露风险高 | 改为环境变量注入，或接入 OAuth2 Resource Server |
| API Key 无失效/轮换机制 | 一旦泄露难以止损 | 增加密钥有效期、多密钥管理或 JWT 方案 |
| MCP Streamable-HTTP 端点复用业务鉴权 | MCP 协议客户端通常不带 `X-API-Key` | 为 `/mcp` 增加独立的 Bearer Token 或 OAuth2 支持 |
| H2 Console 默认开启 | 生产环境存在 SQL 注入/数据暴露风险 | 生产 profile 中显式关闭 `spring.h2.console.enabled` |
| 无请求限流/防重放 | 容易被恶意刷接口 | 引入 Bucket4j 或网关层限流 |

### 6.2 代码与架构问题

| 问题 | 位置 | 建议 |
|------|------|------|
| `nextOrderNo()` 实现存在并发和准确性问题 | `SalesOrderService:147` | 使用数据库序列（如 PostgreSQL `serial`）或分布式 ID（Snowflake） |
| `parseLineItems()` 中对 `NumberFormatException` 的捕获范围过大 | `EnterpriseMcpServer:185-206` | 细化 try-catch 范围，或提前做正则校验 |
| `ApiKeyAuthenticationFilter` JSON 转义不完整 | `ApiKeyAuthenticationFilter:70-72` | 使用 Jackson `ObjectMapper` 或 `StringEscapeUtils` 做标准 JSON 转义 |
| 缺少 API 版本控制的长期设计 | `/api/v1` | 若后续拆分微服务或升级 V2，需提前规划 DTO 兼容性 |

### 6.3 测试与质量

| 问题 | 建议 |
|------|------|
| 测试数量仅 5 个 | 补充各层的单元测试（Service、MCP Server、报价计算逻辑） |
| 缺少 MCP 协议的端到端测试 | 增加 `MockMvc` 或 `TestRestTemplate` 对 `/mcp` initialize + tools/list 的验证 |
| 缺少数据库约束测试 | 验证唯一索引冲突、外键级联等场景 |
| 无静态代码分析 | 建议接入 SpotBugs、Checkstyle 或 SonarQube |

### 6.4 生产化建议

| 项 | 建议 |
|----|------|
| 数据库 | 迁移至 PostgreSQL，保留 Flyway 脚本，增加 `flyway-database-postgresql` |
| 日志与可观测性 | 引入 OpenTelemetry + Micrometer，对接 Prometheus/Grafana |
| 容器化 | 补充 `Dockerfile` 和 `docker-compose.yml`，支持一键启动 |
| 多模块拆分 | 若业务持续增长，建议按 catalog/customer/sales 拆分为 Maven 子模块或 Spring Modulith |
| API 文档 | 引入 SpringDoc OpenAPI 自动生成 Swagger UI |
